  - **Scoped optimistic updates are a reviewer-favorite catch.** Original implementation bumped `fetched_at` for every 200 response; the 6h throttle means `{status: 'already_fresh'}` is common (user pulls within the throttle window) and showing "just now" in that case lies to the user until the next real refresh. Rule: only optimistically mutate state on the branch that will definitely produce a server-side change — branches that represent "server rejected / no-op" must leave UI state alone.
  - **Derive a stable subscription key from a sorted-ids string for `postgres_changes` filters with `in.()` operator.** A new `setLinks` reference triggers the effect even when the underlying ids are unchanged, causing a resubscribe storm. `useMemo(() => links.map(l => l.id).sort().join(','), [links])` gives a string key that only changes when the SET of ids actually changes. Dep the effect on the string, not the array — React compares by reference for arrays but by value for strings.
