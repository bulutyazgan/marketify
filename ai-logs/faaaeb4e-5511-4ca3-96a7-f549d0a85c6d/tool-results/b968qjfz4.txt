- **Learnings for future iterations:**
  - The AuthUser synthesis pattern from US-033 carried over verbatim (inline id + trimmed form fields + new ISO timestamps + null deleted_at, with `email: null` for creator role). Keep this shape the canonical post-signup AuthUser construction until the edge function contract evolves to return DB-generated timestamps.
  - **Full-screen loading-state swaps need their own `accessibilityLiveRegion` on the new screen's header.** Android's AccessibilityNodeInfo does not fire live-region events for subtree swaps — only for content changes within a mounted node (the US-033 inline-error pattern relies on that). When flipping between distinct UI screens inside a single route (form → loading → navigate), the incoming screen's primary status Text/View should carry `accessibilityRole='header' + accessibilityLiveRegion='polite'` to announce the transition. iOS VoiceOver re-reads the screen on mount so this also helps but is not strictly required there.
  - **Avoid the `didSucceed` finally-guard on navigation-triggering submits.** The temptation is real: `router.replace` unmounts the screen, so `setSubmitting(false)` in `finally` "could flash the form for a frame". But `router.replace` is synchronous enough under React Native that no flash materialized in testing, and React suppresses state updates on unmounted components. Meanwhile the guard makes the form unrecoverable if the navigation itself throws (rare but real — deep-link intercepts, a crash in the destination layout, a back-gesture race). Trust the reconciler; always call `setSubmitting(false)` in `finally`.
  - The existing `mobile_type_keys` first-char-drop gotcha (#101) also drops chars mid-string, not just the first char — observed `u034_both_1719`→`u34_both_1719`, `tiktok`→`tktok`, `instagram`→`istagram` in a single two-handle fill. Tests that don't care about handle *accuracy* (e.g. this loading-state screenshot just needs both fields non-empty) can tolerate the drops; tests that DO care (e.g. duplicate-username 409, real Apify scrape) must verify via `mobile_list_elements_on_screen` after each type and retype via the Select All menu.
